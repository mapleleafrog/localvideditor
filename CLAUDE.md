# soranji-vfx ‚Äî Project Context

## What this is
A Remotion (React ‚Üí MP4) VFX pipeline for a wedding walk-in video set to "Soranji" (~120 BPM). HTML/CSS-authored animation, rendered frame-accurate to MP4. Output: 1920√ó1080, 30fps. Scaffolded directly into this folder (the project root holds `package.json`; the source specs live in `specs/`).

## Creative direction (locked)
2D animation that reads as having depth via **shadows, bevel, and z-mapping/parallax** ‚Äî NOT real 3D. **Pixel-art** is a featured aesthetic (MapleStory-style sprites, stepped/quantized motion, CRT/scanline looks). True-3D effects (Three.js: card stack, turntable, 2.5D depth-map) are intentionally left as typed `todo` stubs.

## Stack (pinned, identical versions ‚Äî no `^`)
- remotion + @remotion/cli + @remotion/transitions + @remotion/media-utils + @remotion/paths + @remotion/shapes + @remotion/gif ‚Äî all **4.0.472**
- React 19, TypeScript. No UI/state libraries (Tailwind was stripped from the scaffold).
- `@remotion/media-utils` is wired for audio **duration** (`getAudioDurationInSeconds`, sizes the timeline to the song). Beat is still a frame-derived metronome (configurable BPM/offset) ‚Äî true FFT audio-reactive (`useAudioData`/`visualizeAudio`) is still `todo`.

## Architecture
- `src/effects/` is a data-driven registry grouped by category.
  - `catalog.ts` ‚Äî the full ~110-effect catalog as metadata (the single source of truth). Every row starts `status:'todo'`.
  - `motions.ts` ‚Äî single-layer animations; imports the catalog, overrides the implemented ids to `status:'ready'` with a `style(ctx) => CSSProperties`. Merges in the depth + pixel families.
  - `transitions.ts` ‚Äî footage A‚ÜíB; built-in presentations + custom CSS presentations + shipped shader presentations, the rest `todo`.
  - `depth.ts` ‚Äî the 2.5D toolkit: `depthShadow(z)`, `depthScale(z)`, `bevel()` + depth motions.
  - `pixel.ts` ‚Äî pixel-art motions (quantized/stepped).
  - `presentations.tsx` ‚Äî custom CSS `TransitionPresentation` components (dip, flash, masks, zoom look-alikes, pixel reveals).
  - `index.ts` ‚Äî `getMotion(id)` / `getTransitionPresentation(id, params)` (always safe-fallback, never throw) + `buildCreditsMarkdown()` + `readyMotions()` / `readyTransitions()`.
- Effects carry `tier` (Core/Ext/Adv), `engine` (css/canvas/webgl/three/either), `status` (ready/todo), `tags`, and `license`/`credit`.
- `Layer.tsx` is the single frame‚Üíctx boundary: reads `useCurrentFrame()`/`useVideoConfig()`, computes `progress`/`t`/`beat`/`z`, mount-gates the layer, composes centering, applies the motion's CSS, and runs per-overlay **enter/exit transitions** ‚Äî an element-scoped set `fade`/`slide{Left,Right,Up,Down}`/`zoom`/`pop`/`rotateIn`/`spin`/`blurIn`/`flash`/`wipe`/`iris`/`typewriter` (`ioPart()`), ramped over `enter`/`exitDurationInFrames` and folded into one combined opacity + transform + filter + clip-path. These are layer-scoped reveals (distinct from the clip library's full-frame A‚ÜíB presentations, which assume two full-frame scenes). Set per overlay in the Inspector's **Transitions** section. Layers/clips also carry an optional **`strength`** (effect intensity ‚Äî `scaleStrength()` in `compose.ts` dials any motion's transform/filter/opacity toward identity, generic so it needs no per-formula changes; backgroundColor-type effects don't scale) and overlays an optional **`loop`** (motion `progress` sawtooths over `windowInFrames` instead of running once).
- **Easing curves** (`src/effects/easing.ts`): pure cubic curves `EASINGS = { linear, easeIn, easeOut, easeInOut, easeOutIn }` + `ease(name, t)` helper. Applied in three places: (1) **clip transitions** ‚Äî `clip.transitionEasing` feeds `linearTiming({ easing })` so the A‚ÜíB presentation accelerates/decelerates; (2) **layer enter/exit ramps** ‚Äî `enterEasing`/`exitEasing` curve the reveal; (3) **per-effect progress** ‚Äî each stacked motion's `progress` is run through its own easing.
- **Per-effect settings** (`motionParams: MotionParam[]`, index-aligned with `overlay.motions`): each stacked motion gets its own `loop` / `strength` / `easing` instead of one layer-wide setting. A per-effect value falls back to the layer-level `loop`/`strength` when unset. So ticking "loop" on one effect no longer loops all of them. The data model keeps `motions: string[]` (back-compat, Studio-friendly) and rides a parallel `motionParams` array. Edited per-chip in the Inspector's Effects section (each `.fx-item` block: Strength slider ¬∑ Easing select ¬∑ Loop checkbox ¬∑ remove).
- **Strength UX** (`editor/src/lib/strength.ts` `respondsToStrength(id)`; `Inspector.tsx`; `Slider` gained a `disabled` prop, greyed via `.sld-disabled`): since `scaleStrength` only dials transform/filter/opacity toward identity, a motion whose style is e.g. `left`/`top` percentages (motionPath/arcMove/orbit) or a raw `backgroundColor` doesn't respond to strength at all ‚Äî dragging its slider would silently do nothing. `respondsToStrength` detects this once per motion id (cached) by dense-sampling the motion's style fn across a window and diffing it against its own `scaleStrength(¬∑, 0.4)` output; a mismatch on any CSS key means it responds. The per-effect Strength slider (Inspector Effects section) and the clip Motion section's strength slider grey out + show "no intensity for this effect" when the id doesn't respond. The overlay Transform/Layer section also gained a layer-wide **"Strength (all effects)"** slider (`o.strength`, 0‚Äì2) ‚Äî and the per-effect slider's displayed value falls back to it (`p.strength ?? o.strength ?? 1`) when unset, so an unset per-effect slider shows the INHERITED layer value instead of always reading 1.0 regardless of the layer-wide setting. Known pre-existing quirk (not fixed here): `scaleStrength` lerps opacity toward 1, so additive low-opacity fx (beatFlash/grainLoop/crtScanlines) go fully opaque at strength 0 instead of vanishing.
- `Mushroom.tsx` renders animated GIF sprites via `@remotion/gif`'s `<Gif>` with crisp `pixelated` scaling.

## Conventions (non-negotiable)
- Frame-driven only. No @keyframes, setInterval, rAF, GSAP ticker, or R3F useFrame.
- Time-window layers by mount-gating in `Layer` (return null outside range). Do NOT wrap beat-reactive layers in a frame-rebasing `<Sequence>` ‚Äî it desyncs the beat.
- `t` is ABSOLUTE composition seconds so loops + beat stay on the song grid.
- All assets via `staticFile()`. No remote URLs in components.
- BPM via `helpers.ts#beatKick(t, bpm, exp, offsetSec)` only ‚Äî beat is frame-derived (no FFT). `project.bpm` + `project.beatOffsetInFrames` set the metronome; `Layer`/`Timeline` thread them into every `beatKick` call so beat-reactive motions lock to the song's tempo/downbeat.
- New effects = catalog entry + (if ready) an implementation override. Scenes reference effects by id.
- Composition length must equal footage content (transitions overlap; total = Œ£sequenceDurations ‚àí Œ£transitions).

## Shader transitions caveat
The 7 shader presentations are `engine:"webgl"` (HTML-in-canvas). They **render correctly** via `npx remotion render` / Lambda, but their **Studio preview** needs Chrome 149+ with `chrome://flags/#canvas-draw-element` enabled. They are excluded from the preview-safe `TransitionGallery`. The 3 light-leak transitions are also `engine:"webgl"`, but they are **plain** WebGL (a `<canvas>` shader from `@remotion/light-leaks`, NOT HTML-in-canvas) ‚Äî they preview with no flag, yet are still tagged `webgl` so they stay out of that gallery (keeping the reel WebGL-free, consistent with the shaders). A CLI render of them needs a WebGL backend (`npx remotion render ‚Ä¶ --gl=angle`; the editor's in-app render plugin already sets `chromiumOptions.gl:"angle"`). Everything else previews with no flags.

## Make a video (Timeline)
The `Timeline` composition renders ANY video from a `project` config (Zod-schema'd props): a **clip track** (image/video clips joined by transitions) + positioned **overlays** (text/image with stacked effects) + a **background**. Effect ids resolve against `src/effects`, so new effects work immediately ‚Äî one registry, exact preview, render-grade. Files: `src/timeline/schema.ts` (`projectSchema`, `SAMPLE_PROJECT`), `src/timeline/Timeline.tsx`; sample configs in `projects/`.
- **Edit:** `npm run dev` ‚Üí Studio opens `Timeline` with a live props form + scrubber (or hand-edit a `projects/*.json`). Drop photos/clips in `public/media/`, reference as `media/<file>`.
- **Render:** `npm run render:timeline` (sample) or `npx remotion render Timeline out/video.mp4 --props=./projects/your.json`.
- Duration is computed from the config (`calculateTimelineMetadata`, async: max(Œ£clips ‚àí Œ£transitions, overlay end, **audio end** via `audioEndFrames`), so the video lasts as long as the song) ‚Äî UNLESS `project.durationInFrames` is set (>0), which **fixes** the length and caps how far clips/overlays can be extended (Canvas tab ‚Üí Project duration). Overlays take `motions: string[]`, composed via `composeStyles` (`src/effects/compose.ts`).
- **Per-character text animation:** text overlays carry an optional `textAnimation` (`charFadeUp` / `charBlurReveal` / `typewriterChar` / `wordHighlight`) + `textAnimationStagger` (frames). When set, `Timeline.tsx` renders the string as per-**grapheme** `<span>`s (`Intl.Segmenter`, so emoji / combining kana don't shatter ‚Äî falls back to `Array.from`), each revealed on the frame clock (local `f = frame ‚àí from`, offset by `index √ó stagger`, ramped over 12 frames) ‚Äî a **renderer feature, not a `style(ctx)` motion** (a single CSS formula can't emit per-child styles, so this lives in `Timeline.tsx`, not the effects registry / `portable.ts`). `typewriterChar` hides via opacity only (keeps the editor's on-canvas size-measure stable). Edited in the Inspector's Text section; unset / `none` = static text, so existing projects are byte-identical.
- **Japanese fonts:** text overlays carry an optional `fontFamily` id (`notoSerifJP` / `zenMaruGothic` / `shipporiMincho` / `notoSansJP`; else `default` = monospace), resolved by `src/timeline/fonts.ts` via `@remotion/google-fonts`. Loading is **lazy + memoized** (only a project that actually resolves a JP font fetches it ‚Äî `SoranjiSample` / the galleries pay nothing) and **render-safe** (`loadFont` gates each subset chunk on `delayRender`, so preview == MP4). A used JP family pulls ~120 `fonts.gstatic.com` subset chunks (weight 700 only) ‚Äî a network dependency at render time; the static font-metadata imports also grow the editor bundle (~0.65 ‚Üí ~1.4 MB). `resolveFontFamily(unset/unknown)` ‚Üí `monospace`, so existing projects are unchanged. Picked in the Inspector's Text section.
- **Two ways to edit the same `project` schema:** the Studio props form (`npm run dev`) **or** the drag-and-drop editor below (`npm run editor`). Both render via the same `Timeline` composition + CLI pipeline.

### Audio & beat-sync
- `projectSchema.audio` is an array of soundtrack tracks (`src`, `volume`, `from`, `trimBefore`, `trimAfter`, `loop`); `Timeline.tsx`'s `SoundtrackLayer` renders each as `<Audio>` inside a `<Sequence from={track.from}>` (preview + render). Non-looping tracks **extend** the timeline via `audioEndFrames`; looping tracks fill it and never define it.
- **Beat-sync (not FFT):** `project.bpm` + `project.beatOffsetInFrames` drive the synthetic beat clock. Apply a beat-reactive motion (`beatPulse`/`beatShake`/`beatFlash`/`beatColorCycle`/`beatZoomCut`) to any clip/overlay and it pulses on that tempo ‚Äî set BPM to the song and nudge the offset so the kick lands on the downbeat. Drop `.mp3/.wav/.m4a` in `public/media/`. Real waveform-reactive beat (bass/loudness) is still `todo` (`bassWarp`, `audioBars`, `waveform`).
- **Editor:** the Library **Audio tab** (`editor/src/components/AudioPanel.tsx`) sets BPM/offset and adds/edits soundtrack tracks. It's a full import surface ‚Äî **drag audio onto it or "+ Add audio files"** uploads to `public/media/<project>/`, adds each as a track, and refreshes the picker. Audio is excluded from the **Assets** grid (`/api/media` returns `{ assets, audio }` split), so dropping audio on the Assets panel (or timeline/canvas) routes it straight to a soundtrack track instead of vanishing ‚Äî the Assets toast points to the Audio tab. `Preview` reads audio lengths in-browser to size the `<Player>`.

## Soranji Studio ‚Äî the drag-and-drop editor (`editor/`, `npm run editor`)
A real timeline NLE (Canva/CapCut-style) built on `@remotion/player`, hosted by a Vite app in `editor/`. **The live preview IS the `Timeline` composition** that renders to MP4 ‚Äî exact WYSIWYG. **Registry-driven:** every effect/transition picker reads `src/effects` via `editor/src/lib/effects-bridge.ts`, so a motion added to `portable.ts`+`catalog.ts` appears in the editor with zero editor edits (the whole point ‚Äî scales with the library). Full how-to in `EDITOR_GUIDE.md`.
- **Layout:** Topbar (project-name field ¬∑ undo/redo ¬∑ ‚å® shortcuts ¬∑ Save/Export/Import/Reset/Delete ¬∑ ‚è∫ Render) ¬∑ Library (Effects/Transitions/Assets/Audio/Canvas) ¬∑ Preview (Player + on-canvas drag/scale/rotate + click-select) ¬∑ Inspector (per-item props incl. effect multi-select) ¬∑ Timeline (clip track + per-overlay lanes, drag-retime/resize, scrub, ‚ñ≤‚ñº / drag-to-reorder lanes = z-order, **drop media to add a clip/audio**, **lanes scroll vertically** ‚Äî the gutter is JS-synced to the lane scroll and the time-ruler is pinned).
- **Right-click context menus** (`ContextMenu.tsx`, a single instance mounted in `App.tsx`; store `ctxMenu`/`openCtxMenu`/`closeCtxMenu`): right-click a timeline clip block, overlay block, gutter row, or canvas node (`.cv-hit`/selected `.cv-body`) for a fixed-position, viewport-clamped menu (split/duplicate/copy/paste/delete, plus bring-forward/send-backward/move-to-top/bottom for overlays, plus "Set motion‚Ä¶"/"Set transition ‚Üí next‚Ä¶"/"Add effect‚Ä¶" opening the Effect Browser below, plus Flip horizontal/vertical and the quick-transform actions below) operating on the just-selected target. `openCtxMenu` captures the playhead frame from the caller BEFORE selecting (selecting an overlay can auto-seek the player), so "Split at playhead" stays accurate. Every menu-trigger handler calls `stopPropagation` so opening a new menu never races the previous one's close-on-second-right-click listener; right-click elsewhere still falls through to the native browser menu. Drag handlers guard `e.button !== 0` so a right-click never starts a block/handle/canvas-gesture drag. The list keeps growing, so `.ctx-menu` is capped at `max-height: calc(100vh - 16px)` with `overflow-y: auto` so Delete stays reachable on short viewports.
- **Quick transform actions** (overlay context menu, `ContextMenu.tsx`): "Rotate 90¬∞ CW/CCW" (`patchOverlay(i,{rotation: normAngle(o.rotation¬±90)})`, `normAngle` = the same `((deg+180)%360+360)%360-180` formula `CanvasOverlay.tsx`'s rotate gesture uses, so both stay in the same [-180,180] convention), "Center on canvas" (`x:50,y:50`), "Reset transform" (`x:50,y:50,scale:1,rotation:0,flipX:undefined,flipY:undefined` ‚Äî `undefined` clears the field rather than writing a stale `false`), and "Fit to frame"/"Native size" for **image overlays only** ‚Äî async: resolves the src the same way `AssetsPanel.tsx` does (`staticFile()` or http passthrough), reads `imageNaturalSize`, and re-checks the overlay still exists with the same `src` after the await (it may have been deleted/replaced while the image was loading) before patching. An unreadable image (bad src, or a non-image asset) flashes "Couldn't read image size" and does NOT patch `width:0`. `fx` overlays disable Rotate/Center/Reset/Fit/Native (they have no x/y/scale/rotation) but keep Flip enabled; Fit/Native are disabled for non-image overlays and for clips (clips fill the whole frame, so those two items are always disabled there, kept only for a consistent menu shape).
- **Pop-up Effect Browser** (`EffectBrowser.tsx`, mounted in `App.tsx`; store `browser`/`openBrowser`/`closeBrowser`): a modal card grid over the full motion or transition registry ‚Äî search + category select + top-tag chips (AND-composed), grouped-by-category when unfiltered. Transition cards are static (no live preview yet) with a ‚ö° badge on `engine:"webgl"` ones. Entry points: Library's "‚äû Browse all‚Ä¶" (Effects tab ‚Üí overlay-add/clip-motion by selection kind; Transitions tab ‚Üí clip-transition), Inspector's "Browse‚Ä¶" next to the motion adder and in the clip Motion/Transition sections, and the context-menu items above. Picking in `overlay-add` mode appends and stays open (‚úì badge on applied cards, re-click is a no-op); `clip-motion`/`clip-transition` patch and close. The modal re-validates its target every render (closes itself if the clip/overlay vanished) and every pick reads the store fresh (`useEditor.getState()`), so it never acts on stale props. **The modal is a keyboard-focused context:** `App.tsx`'s global keydown checks `browser` first ‚Äî Escape closes it, every other key (Space/Delete/Ctrl+D/K/S/Z‚Ä¶) is swallowed before the rest of the shortcut map runs.

Edits flow store‚ÜíPlayer live.
- **Flip horizontal/vertical** (`schema.ts` `flipX`/`flipY` on both `clipSchema` and `overlaySchema`; `Layer.tsx`; `Timeline.tsx`): mirrors a layer's/clip's content in place without inverting its motion's direction or displacing it. For **content overlays** (text/image/video ‚Äî `Layer` gets `children`), `Layer.tsx` wraps `children` in an INNER content wrapper carrying only the flip transform with `transformOrigin: "center center"` ‚Äî deliberately kept off the outer motion-transform chain, since bottom/top-anchored motions (squashStretch/pendulum/eightBitHop/swayLoop/pixelWindSway) set `transformOrigin: "bottom/top center"` there, and a flip on that chain would mirror about that origin and displace the layer by a full height. For **fx full-frame layers** (no `children` ‚Äî the motion paints the frame via `backgroundImage` on the OUTER Layer div, so an inner wrapper would mirror an empty div = no-op), the flip is instead folded into the outer transform (`Layer.tsx` branches on `hasChildren`) ‚Äî safe because atmospheric fx motions (weddingPetals/bokehLights/godRays/lensFlare/lightLeakWarm/sakuraPetals/sparkleField) are backgroundImage-based with the default center origin. For clips, flip applies directly to the `<Img>`/`<OffthreadVideo>` media element (no wrapper needed ‚Äî it carries no other transform). Editable via Inspector checkboxes (Transform / clip fields, available for `fx` layers; uncheck clears the field to `undefined` rather than persisting `false`) or the right-click context menu's Flip horizontal/vertical toggles (‚úì-prefixed when active).
- **Storyboard view** (`editor/src/components/Storyboard.tsx`; **Edit | Storyboard** toggle in the Topbar, `view` state in the store): a clip-tied card board ‚Äî each card IS a clip (footage thumbnail / video first-frame / placeholder + editable duration + **Label**/**Notes** + `src` + inline "‚Üí next" transition picker; native-HTML5 drag-to-reorder via `reorderClip`, ‚óÄ‚ñ∂ buttons, "+ Add shot", click-to-select carries into Edit). Clips gained two optional render-ignored fields `label`/`note` (`schema.ts`); Storyboard ‚áÑ Edit share `project.clips`. Overlays/titles/fx are NOT storyboard cards (edited in the Edit tab).
- **State:** zustand + zundo (`editor/src/store.ts`) ‚Äî only `project` is undoable (`partialize`), so transient fields (`selection`/`playhead`/`zoom`/`clipboard`/`toast`/`showShortcuts`) never pollute history; a throttled `handleSet` (600ms, leading) coalesces rapid drags/scrubs/typing into ONE undo step (so Ctrl+Z reverts a whole motion, not a few px); autosaves to `localStorage`, seeded from `projects/sample.json`.
- **In-app render + persistence** (`editor/render-plugin.ts`, a Vite dev-server plugin): `POST /api/render` (bundle once ‚Üí `selectComposition` ‚Üí `renderMedia`, streamed NDJSON progress; tuned for the local machine ‚Äî `concurrency` = all CPU cores, GPU rendering via `chromiumOptions.gl:"angle"`, and max-quality H.264 `jpegQuality:100`/`crf:16` with software x264; all three adjustable from the Topbar **‚öô Render settings** popover and sent in the POST `options`) ¬∑ `POST /api/save-project` (‚Üí `projects/*.json`) ¬∑ `POST /api/upload?name=&project=` (writes bytes to `public/media/<project>/`, suffixes on collision) ¬∑ `POST /api/delete-project` (removes the JSON + media folder) ¬∑ `GET /api/media?project=` (root `public/` + flat `public/media/` + that project's folder, split into `{ assets, audio }`). Same renderer/output as `npx remotion render Timeline --props=<json>`.
- **Export modes** (format picker by the Render button ‚Üí `options` in the POST body): `mp4` (H.264) ¬∑ `alpha` (transparent ProRes 4444 `.mov`, `yuva444p10le`) ¬∑ `overlays` (alpha + clip track dropped = just the VFX/titles). The alpha modes set `background:none` (Timeline renders a transparent root when `background.type==='none'`) and overlays-only empties `clips`, so you export elements to composite + beat-cut in DaVinci/Premiere. Verified: `yuva444p` ProRes carries a real alpha channel.
- **Editor has its own tsconfig** (`editor/tsconfig.json`, ESNext/Bundler) ‚Äî the root `tsconfig.json` excludes `editor/`. Typecheck the editor with `npx tsc -p editor/tsconfig.json`. Editor-only deps (`@remotion/player`, `react-moveable`, `selecto`, `zustand`, `zundo`, `vite`, `@vitejs/plugin-react`, `@types/react-dom`) are pinned in `package.json`.
- **Not yet:** multi-track/marquee multi-select, keyframes, `<Audio>` + real beat-sync. (Single-selection model; selecto deferred in favor of click-select + reorder.)

## No-npm Retro Portal (`index.html`)
A self-contained synthwave/arcade playground at the project root ‚Äî **double-click to open, no npm, no server**. It is a live preview/composer (CSS driven by `requestAnimationFrame`), NOT the render pipeline. **The portal's motion effects + `composeStyles` are SINGLE-SOURCED**: it loads `public/portal/effects.bundle.js` (generated from `src/effects/portable.ts` by `npm run gen:portal`) and drives its MOTIONS tab + stacking from `window.SoranjiEffects` ‚Äî it declares zero motion formulas of its own, so a formula added to `portable.ts` shows up here and in the render with no hand-mirroring. `npm run check:portal` (or `check-portal.bat`) guards against a stale bundle, and `predev`/`prerender:timeline` regenerate it automatically. Stack multiple motions at once (transforms + filters concatenated, opacity multiplied). The TRANSITIONS tab keeps its own CSS approximations (‚ö°) inline (transitions are out of the single-source scope).
